## Pattern 1: 0/1 Knapsack

Given items with weights and values, and a knapsack capacity W, maximize the total value without exceeding W. Each item can be taken at most once.

**State**: `dp[i][w]` = max value using first i items with capacity w

**Recurrence**: `dp[i][w] = max(dp[i-1][w], dp[i-1][w - weight[i]] + value[i])`

```python
def knapsack_01(weights, values, capacity):
    n = len(weights)
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for w in range(capacity + 1):
            dp[i][w] = dp[i - 1][w]  # don't take item i
            if weights[i - 1] <= w:
                dp[i][w] = max(dp[i][w],
                               dp[i - 1][w - weights[i - 1]] + values[i - 1])

    return dp[n][capacity]
```

### Space-Optimized (1D)

Since each row only depends on the previous row, use a single array and iterate **right to left** to avoid using updated values.

```python
def knapsack_01_optimized(weights, values, capacity):
    dp = [0] * (capacity + 1)

    for i in range(len(weights)):
        for w in range(capacity, weights[i] - 1, -1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])

    return dp[capacity]
```

**Time**: O(n × W). **Space**: O(W).

### Unbounded Knapsack Variant

When items can be used unlimited times, iterate **left to right** in the 1D version.

```python
def knapsack_unbounded(weights, values, capacity):
    dp = [0] * (capacity + 1)
    for w in range(1, capacity + 1):
        for i in range(len(weights)):
            if weights[i] <= w:
                dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[capacity]
```

### Subset Sum (Knapsack Variant)

Can you select a subset that sums to exactly target?

```python
def can_partition(nums, target):
    dp = [False] * (target + 1)
    dp[0] = True
    for num in nums:
        for t in range(target, num - 1, -1):
            dp[t] = dp[t] or dp[t - num]
    return dp[target]
```

## Pattern 2: Longest Common Subsequence (LCS)

Find the length of the longest subsequence common to two strings.

**State**: `dp[i][j]` = LCS length of `text1[:i]` and `text2[:j]`

**Recurrence**:
- If `text1[i-1] == text2[j-1]`: `dp[i][j] = dp[i-1][j-1] + 1`
- Otherwise: `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`

```python
def longest_common_subsequence(text1: str, text2: str) -> int:
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]
```

```go
func longestCommonSubsequence(text1, text2 string) int {
    m, n := len(text1), len(text2)
    dp := make([][]int, m+1)
    for i := range dp {
        dp[i] = make([]int, n+1)
    }
    for i := 1; i <= m; i++ {
        for j := 1; j <= n; j++ {
            if text1[i-1] == text2[j-1] {
                dp[i][j] = dp[i-1][j-1] + 1
            } else {
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
            }
        }
    }
    return dp[m][n]
}
```

**Time**: O(m × n). **Space**: O(m × n), optimizable to O(min(m, n)).

### Reconstructing the LCS

```python
def lcs_string(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    # Backtrack to find the actual subsequence
    result = []
    i, j = m, n
    while i > 0 and j > 0:
        if text1[i - 1] == text2[j - 1]:
            result.append(text1[i - 1])
            i -= 1
            j -= 1
        elif dp[i - 1][j] > dp[i][j - 1]:
            i -= 1
        else:
            j -= 1
    return "".join(reversed(result))
```

## Pattern 3: Longest Increasing Subsequence (LIS)

Find the length of the longest strictly increasing subsequence.

### O(n²) DP Solution

```python
def length_of_lis(nums):
    n = len(nums)
    dp = [1] * n  # dp[i] = LIS ending at index i

    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)

    return max(dp)
```

### O(n log n) with Binary Search

Maintain a "tails" array where `tails[i]` is the smallest tail element of any increasing subsequence of length `i + 1`.

```python
import bisect

def length_of_lis_optimal(nums):
    tails = []
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    return len(tails)
```

**Time**: O(n log n). **Space**: O(n).

> **The `tails` array is NOT the LIS itself** — it's a helper structure. To reconstruct the actual subsequence, maintain a predecessor array alongside.

## Pattern 4: Coin Change (LeetCode 322)

Given coin denominations and a target amount, find the minimum number of coins needed.

**State**: `dp[amount]` = min coins to make `amount`

**Recurrence**: `dp[i] = min(dp[i - coin] + 1) for each coin`

```python
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0

    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i and dp[i - coin] != float('inf'):
                dp[i] = min(dp[i], dp[i - coin] + 1)

    return dp[amount] if dp[amount] != float('inf') else -1
```

```go
func coinChange(coins []int, amount int) int {
    dp := make([]int, amount+1)
    for i := range dp {
        dp[i] = amount + 1
    }
    dp[0] = 0

    for i := 1; i <= amount; i++ {
        for _, coin := range coins {
            if coin <= i && dp[i-coin]+1 < dp[i] {
                dp[i] = dp[i-coin] + 1
            }
        }
    }
    if dp[amount] > amount {
        return -1
    }
    return dp[amount]
}
```

**Time**: O(amount × len(coins)). **Space**: O(amount).

### Coin Change II: Count Ways (LeetCode 518)

```python
def change(amount, coins):
    dp = [0] * (amount + 1)
    dp[0] = 1
    for coin in coins:  # iterate coins first to avoid counting permutations
        for i in range(coin, amount + 1):
            dp[i] += dp[i - coin]
    return dp[amount]
```

The order of loops matters: iterating coins in the outer loop counts **combinations**, not permutations.

## Pattern 5: Edit Distance (LeetCode 72)

Find the minimum operations (insert, delete, replace) to transform word1 into word2.

**State**: `dp[i][j]` = edit distance between `word1[:i]` and `word2[:j]`

```python
def min_distance(word1: str, word2: str) -> int:
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i - 1] == word2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = 1 + min(
                    dp[i - 1][j],      # delete
                    dp[i][j - 1],      # insert
                    dp[i - 1][j - 1]   # replace
                )

    return dp[m][n]
```

**Time**: O(m × n). **Space**: O(m × n), optimizable to O(min(m, n)).

## Pattern 6: Matrix Chain Multiplication

Find the most efficient way to multiply a chain of matrices.

**State**: `dp[i][j]` = min cost to multiply matrices i through j

```python
def matrix_chain_order(dims):
    """dims[i] × dims[i+1] are dimensions of matrix i."""
    n = len(dims) - 1
    dp = [[0] * n for _ in range(n)]

    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            dp[i][j] = float('inf')
            for k in range(i, j):
                cost = (dp[i][k] + dp[k + 1][j] +
                        dims[i] * dims[k + 1] * dims[j + 1])
                dp[i][j] = min(dp[i][j], cost)

    return dp[0][n - 1]

# Matrices: A1(10×30), A2(30×5), A3(5×60)
# dims = [10, 30, 5, 60]
# Optimal: (A1 × A2) × A3 = 10*30*5 + 10*5*60 = 4500
```

**Time**: O(n³). **Space**: O(n²). This pattern appears in many interval DP problems.

## Pattern Recognition Cheatsheet

| Problem Pattern | DP State | Example Problems |
|----------------|----------|-----------------|
| Knapsack | `dp[i][w]` = value with items 1..i, capacity w | Subset Sum, Partition Equal |
| LCS | `dp[i][j]` = LCS of s1[:i], s2[:j] | Edit Distance, Shortest Common Supersequence |
| LIS | `dp[i]` = LIS ending at i | Russian Doll Envelopes |
| Coin Change | `dp[amount]` = min coins | Perfect Squares, Combination Sum IV |
| Grid Path | `dp[i][j]` = value at cell (i,j) | Unique Paths, Min Path Sum |
| Interval | `dp[i][j]` = optimal for range [i,j] | Matrix Chain, Burst Balloons |

### Exercise 3: Target Sum (LeetCode 494)

```python
def find_target_sum_ways(nums, target):
    total = sum(nums)
    if (total + target) % 2 or abs(target) > total:
        return 0
    subset_sum = (total + target) // 2

    dp = [0] * (subset_sum + 1)
    dp[0] = 1
    for num in nums:
        for t in range(subset_sum, num - 1, -1):
            dp[t] += dp[t - num]
    return dp[subset_sum]
```

## Key Takeaways

- **0/1 Knapsack** and its variants (subset sum, partition) are among the most common DP patterns
- **LCS** generalizes to edit distance, diff algorithms, and DNA sequence alignment
- **LIS** has an elegant O(n log n) solution using binary search on tails
- **Coin Change** is the unbounded knapsack: outer loop over coins for combinations, over amounts for permutations
- **Edit Distance** is the Swiss army knife of string DP — forms the basis for spell checkers and diff tools
- Learn to map new problems to these six patterns — most interview DP reduces to one of them
